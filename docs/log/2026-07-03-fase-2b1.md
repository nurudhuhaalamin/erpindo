# Log Kerja — Fase 2b-1: Fondasi Langganan & Arus Kas

**Tanggal:** 3 Juli 2026 · **Status akhir:** selesai, merged ke `main`

## Ringkasan

Seluruh mekanik langganan dibangun **tanpa** menunggu akun payment gateway — saat kunci Midtrans/Xendit tersedia, tinggal menyambungkan pembayaran ke mekanik yang sudah teruji ini. Plus laporan arus kas (butir terakhir laporan keuangan MVP).

## Yang dibangun

### Fondasi langganan
- **Paket & batas** (`PLAN_LIMITS` di shared): Trial/Starter (3 pengguna), Business (15), Enterprise (tak terbatas) — ditegakkan server-side saat mengundang anggota (402 + pesan upgrade bila melebihi).
- **Siklus hidup trial via Cron Trigger harian** (`triggers.crons`, handler `scheduled` di Worker): trial habis → status `past_due` + audit log `billing.trial_expired`.
- **Mode baca-saja saat menunggak**: middleware menolak semua permintaan tulis (non-GET) dengan 402 saat `past_due`; membaca data & laporan tetap boleh; `suspended` tetap memblokir semuanya.
- **UI**: banner countdown trial (≤7 hari) dan banner merah baca-saja di shell aplikasi; kartu **Langganan** di Pengaturan (paket, status, batas pengguna).
- `/me` diperkaya `plan` + `trialEndsAt`; var `TRIAL_DAYS_OVERRIDE` untuk suite pengujian.

### Laporan Arus Kas
- `GET /reports/cash-flow?from&to` — metode langsung sederhana: saldo kas awal (akun 1-1000/1-1100 sebelum periode), kas masuk/keluar per keterangan jurnal, perubahan bersih, saldo akhir. Halaman **Arus Kas** dengan pemilih periode + ekspor CSV.

### Perbaikan ditemukan saat pengujian
- Rute `/__scheduled` (pemicu cron di wrangler dev) tercegat lapisan assets SPA — ditambahkan ke `run_worker_first`.

## Validasi (semua hijau)

- Typecheck · 16 unit test · build.
- **Smoke test 80 pemeriksaan** (5 baru): arus kas dicek eksak (saldo awal 50 jt → masuk 3.000.500 → akhir 53.000.500); cron dipicu via `__scheduled` dengan `TRIAL_DAYS_OVERRIDE=0` → status `past_due`; **membaca tetap boleh (200), menulis ditolak (402)**.
