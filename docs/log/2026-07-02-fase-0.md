# Log Kerja — Fase 0: Fondasi Platform

**Tanggal:** 2 Juli 2026 · **Status akhir:** selesai, merged ke `main`

## Ringkasan

Membangun fondasi teknis erpindo dari nol sesuai blueprint `docs/02-rencana-pengembangan.md`: monorepo, autentikasi lengkap, arsitektur multi-tenant (database-per-tenant), RBAC, audit log, design system, dan CI/CD.

## Yang dibangun

### Struktur monorepo (pnpm workspaces)

| Paket | Isi |
|---|---|
| `apps/api` | Worker Cloudflare (Hono ^4): API `/api/*` + penyaji SPA dari assets binding |
| `apps/web` | React 19 + Vite 6 + TanStack Router/Query + Tailwind v4; PWA manifest |
| `packages/shared` | Skema Zod & tipe kontrak API (dipakai FE dan BE) |
| `packages/db` | Skema Drizzle control-plane + sistem migrasi tertanam (append-only, idempotent) |

### Fungsional

- **Autentikasi**: register → provisioning tenant otomatis; login/logout; sesi cookie `HttpOnly+Secure+SameSite` (token di-hash SHA-256 di DB); password PBKDF2-SHA256 100k iterasi (WebCrypto, kompatibel Workers); verifikasi email & reset password via token sekali-pakai berkedaluwarsa; semua sesi dicabut saat password direset.
- **Multi-tenant**: control-plane D1 (`users`, `tenants`, `memberships`, `sessions`, `tokens`, `audit_logs`) + satu database D1 per tenant. Abstraksi `TenantDbProvider` dengan dua mode: `local` (pool binding untuk dev/test) dan `cloudflare` (buat DB via REST API + query via HTTP) — kode modul bisnis tidak perlu tahu modenya.
- **RBAC**: peran owner/admin/viewer; middleware `requireAuth` + `requireTenantRole(minRole)`; pemeriksaan keanggotaan selalu di sisi server.
- **Undangan tim**: Owner/Admin mengundang via email (token invite), penerimaan memvalidasi kecocokan alamat email.
- **Audit log**: register, login (sukses/gagal), verifikasi, reset password, undangan, perubahan pengaturan.
- **Rate limiting** KV pada register/login/forgot-password.
- **Mailer abstraksi**: console (dev) / Resend (bila `RESEND_API_KEY` diset).
- **Web**: landing, daftar, masuk, verifikasi, lupa/reset password, halaman undangan, shell aplikasi responsif (sidebar desktop, menu mobile), dashboard, pengaturan (profil perusahaan → ditulis ke DB tenant; anggota tim + undang), dark mode, komponen UI dasar (Button, Input, Select, Card, Alert, Badge, Toast, Spinner).
- **CI/CD**: `.github/workflows/ci.yml` — typecheck + unit test + build + smoke test end-to-end pada tiap PR; job deploy ke Cloudflare berjalan otomatis di push `main` bila secret `CLOUDFLARE_API_TOKEN` tersedia (di-skip dengan aman bila belum).

## Validasi (semua hijau)

- `pnpm typecheck` — 4 workspace lulus.
- `pnpm test` — 11 unit test (skema Zod, PBKDF2/token/hash).
- `pnpm build` — bundle web 402 kB (121 kB gzip).
- `pnpm smoke` — **22 pemeriksaan end-to-end** terhadap `wrangler dev` (D1 & KV lokal): register→verify→login→tulis/baca DB tenant→undang→terima undangan→RBAC viewer ditolak menulis→**non-anggota ditolak mengakses tenant lain (bukti isolasi)**→logout mencabut sesi.
- Verifikasi visual Playwright/Chromium: landing, form daftar, dashboard, pengaturan, tampilan mobile 390px — semuanya dirender benar; registrasi lewat UI sungguhan berhasil sampai dashboard.

## Keputusan teknis yang diambil

1. **Migrasi tertanam sebagai konstanta TS** (bukan file .sql) agar Worker dapat mem-provisi & meng-upgrade skema tenant saat runtime tanpa filesystem.
2. **PBKDF2 (WebCrypto) alih-alih Argon2/scrypt murni JS** — native & cepat di Workers, tidak menabrak batas CPU.
3. **Pool binding tenant untuk mode lokal** — binding Workers tidak bisa dinamis; pool `TENANT_DB_1..5` menyimulasikan provisioning nyata di dev/test, sementara produksi memakai D1 dinamis via REST API. Antarmuka sama (`SqlExecutor`).
4. **Smoke test sebagai gerbang CI** — pengujian integrasi berjalan pada runtime workerd asli (bukan mock), memberi kepercayaan tinggi dengan setup sederhana.
