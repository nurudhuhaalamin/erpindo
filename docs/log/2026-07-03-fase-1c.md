# Log Kerja — Fase 1c: Laporan Keuangan & Dashboard

**Tanggal:** 3 Juli 2026 · **Status akhir:** selesai, merged ke `main`

## Ringkasan

Pelengkap MVP: laporan Laba Rugi & Neraca serta dashboard berisi angka sungguhan — semuanya dihitung langsung dari jurnal terposting (satu sumber kebenaran dengan buku besar).

## Yang dibangun

- **API laporan** (`apps/api/src/routes/reports.ts`):
  - `GET /reports/income-statement?from&to` — pendapatan & beban per akun untuk periode, laba/rugi bersih.
  - `GET /reports/balance-sheet?asOf` — aset/kewajiban/ekuitas per tanggal; **Laba (Rugi) Berjalan otomatis masuk ekuitas** sehingga neraca selalu seimbang; flag `balanced`.
  - `GET /dashboard` — kas & bank (saldo akun 1-1000/1-1100 dari jurnal), penjualan bulan berjalan (+jumlah faktur), piutang & hutang belum lunas, nilai persediaan.
- **Web**: halaman **Laba Rugi** (pemilih periode, default bulan berjalan) dan **Neraca** (pemilih tanggal, badge seimbang); **Dashboard** kini menampilkan 5 kartu angka nyata + panduan "mulai dari sini" dengan tautan alur kerja.

## Validasi (semua hijau)

- Typecheck · 16 unit test · build.
- **Smoke test 61 pemeriksaan** (9 baru), angka diverifikasi eksak terhadap skenario dagang: laba rugi (pendapatan 2.950.000 / beban 300.000 / laba 2.650.000), neraca seimbang dengan baris Laba Berjalan 2.650.000, dashboard (kas 52.999.500, piutang 0, hutang 1.110.000, persediaan 700.000), format tanggal salah ditolak, viewer boleh membaca laporan.
- Verifikasi visual Playwright: dashboard dengan angka hidup; halaman Neraca menampilkan badge "seimbang ✓" (Aset 1.309.500 = Kewajiban 1.159.500 + Ekuitas 150.000).

## Catatan

Dengan ini **seluruh rantai akuntansi MVP tersambung**: transaksi → jurnal otomatis → buku besar → neraca saldo → laba rugi & neraca → dashboard. Ekspor Excel/PDF dan tutup buku menyusul (roadmap Bulan 3 / Fase 2).

## Berikutnya

Sisa MVP Bulan 3: ekspor laporan, kartu stok, aging piutang/hutang, tutup buku; lalu Fase 2 (onboarding self-service, billing langganan, PWA penuh).
