# Log Kerja — Fase 1b: Siklus Penjualan & Pembelian

**Tanggal:** 3 Juli 2026 · **Status akhir:** selesai, merged ke `main`

## Ringkasan

Transaksi dagang inti di atas mesin akuntansi Fase 1a: faktur penjualan/pembelian yang otomatis membuat jurnal dan menggerakkan stok (moving average), plus pencatatan pembayaran. Juga: **deployment produksi ke Cloudflare aktif** via Workers Builds (integrasi Git yang dihubungkan pemilik).

## Yang dibangun

- **Migrasi tenant `0003_commerce`**: `invoices`/`invoice_lines`, `purchases`/`purchase_lines`, `stock_movements`, `stock_levels` (qty + biaya rata-rata per produk×gudang), `payments`.
- **Helper akuntansi reusable** (`apps/api/src/lib/accounting.ts`): `postJournal` (benteng balanced-only untuk semua jalur — jurnal manual kini juga lewat sini), `stockIn` (moving average), `stockOut` (HPP = avg cost, menolak stok kurang), `nextDocNo`, konstanta akun sistem.
- **API commerce** (`apps/api/src/routes/commerce.ts`):
  - **Faktur penjualan** → jurnal otomatis: Dr Piutang; Cr Pendapatan; Cr PPN Keluaran; + Dr HPP / Cr Persediaan sebesar biaya rata-rata; stok keluar. Ditolak bila stok kurang.
  - **Faktur pembelian** → jurnal: Dr Persediaan; Dr PPN Masukan; Cr Hutang; stok masuk + update biaya rata-rata.
  - **Pembayaran** terima/bayar → jurnal kas/piutang atau hutang/kas; validasi tidak boleh melebihi sisa tagihan; status dokumen otomatis `paid` saat lunas.
  - **Level stok** per gudang dengan nilai persediaan.
  - PPN 0/11/12% per dokumen; nomor dokumen berurutan INV-/PB-/PAY-.
- **Web**: halaman Penjualan & Pembelian (satu komponen dua mode — form faktur multi-baris, harga terisi otomatis dari master produk, ringkasan subtotal/PPN/total live, daftar dokumen dengan badge lunas/belum + form pembayaran inline), halaman Stok, seksi navigasi "Transaksi".
- **Deployment**: konfigurasi wrangler dipindah ke root (kompatibel `npx wrangler deploy` bawaan Workers Builds), resource produksi dibuat via connector Cloudflare (D1 control-plane + 5 tenant pool + KV), job deploy GitHub Actions dihapus.

## Validasi (semua hijau)

- Typecheck 4 workspace · 16 unit test · build 130 kB gzip.
- **Smoke test 52 pemeriksaan** (12 baru), siklus penuh: jual tanpa stok ditolak → beli 10 pcs @100rb + PPN (PB-00001, total 1.110.000) → stok 10 @avg 100rb → jual 3 pcs @150rb + PPN (total 499.500) → jual melebihi stok ditolak → stok 7 (nilai 700rb) → **HPP otomatis 300rb** → pembayaran lunas (overpay ditolak) → status `paid` → **neraca saldo tetap seimbang** → viewer ditolak membuat faktur.
- Verifikasi visual Playwright: faktur penjualan diposting lewat UI sungguhan (INV-00001, Rp 499.500).
- Deploy produksi Cloudflare: **Deployment successful** (Workers Builds).

## Keputusan teknis

1. **Moving average di tabel `stock_levels`** (bukan dihitung ulang dari mutasi setiap saat) — cepat dan sederhana; mutasi lengkap tetap tercatat di `stock_movements` untuk audit/kartu stok.
2. **Faktur immutable seperti jurnal** — tidak ada edit/hapus; koreksi lewat dokumen pembalik (retur menyusul di fase berikutnya).
3. **Satu komponen UI untuk penjualan & pembelian** — alur identik, hanya konfigurasi (label, jenis kontak, harga default) yang berbeda.
