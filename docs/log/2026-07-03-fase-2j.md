# Log Kerja — Fase 2j: Batch/Lot & Kedaluwarsa FEFO (Gelombang A-5)

**Tanggal:** 3 Juli 2026 · **Status akhir:** selesai, merged ke `main`

## Yang dibangun

- **Migrasi tenant 0007**: kolom `products.track_expiry` + tabel `stock_lots` (lot, tanggal kedaluwarsa, qty per produk-gudang) + indeks FEFO.
- **Skema shared**: baris faktur menerima `lotNo`/`expiryDate` opsional; produk punya `trackExpiry`.
- **`stockIn`**: bila lot disertakan, qty di-upsert ke `stock_lots` (lot sama digabung).
- **`stockOut`**: setelah stok berkurang, lot dikonsumsi **FEFO** — kedaluwarsa terdekat lebih dulu, tanpa tanggal di akhir; stok tak berlot dibiarkan.
- **Validasi pembelian**: produk berpelacakan **wajib** menyertakan tanggal exp per baris (400 bila tidak).
- **Endpoint `GET /stock-lots`**: daftar lot aktif urut FEFO + `daysToExpiry` + hitungan `expiringSoon` (≤ 30 hari).
- **UI**: checkbox "Lacak lot & kedaluwarsa" di form produk (+ kolom FEFO di tabel + kolom `lacak_exp` di template impor CSV); input Lot/Exp per baris muncul otomatis di form pembelian untuk produk berpelacakan; kartu **Lot & kedaluwarsa** di halaman Stok dengan badge merah (lewat) / kuning (≤ 30 hari) dan peringatan jumlah lot hampir kedaluwarsa.

## Validasi (semua hijau)

- Typecheck · 20 unit test · build.
- **Smoke test 149 pemeriksaan** (10 baru): pembelian produk terlacak tanpa tanggal exp ditolak; pembelian 2 lot (LOT-A exp +10 hari, LOT-B +100 hari); daftar lot urut FEFO; peringatan `expiringSoon` = 1 dengan `daysToExpiry` = 10; jual 3 → LOT-A sisa 2; jual 4 → LOT-A habis, LOT-B terpotong jadi 3; produk tanpa pelacakan bebas lot; viewer boleh membaca; **neraca saldo tetap seimbang**.

## Catatan teknis

- Konsumsi lot murni pencatatan kuantitas (HPP tetap moving average) — sederhana dan konsisten dengan seluruh laporan yang ada.
- Transfer antar gudang belum memindahkan rincian lot (dicatat sebagai keterbatasan; qty lot tetap benar di gudang asal).
