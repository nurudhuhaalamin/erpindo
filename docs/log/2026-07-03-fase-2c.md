# Log Kerja — Fase 2c: 2FA TOTP & Landing Page

**Tanggal:** 3 Juli 2026 · **Status akhir:** selesai, merged ke `main`

## Yang dibangun

### Verifikasi dua langkah (2FA TOTP, RFC 6238)
- **Lib TOTP mandiri** (`apps/api/src/lib/totp.ts`): WebCrypto HMAC-SHA1, base32, toleransi ±1 langkah — tanpa dependensi eksternal, kompatibel semua aplikasi authenticator.
- **Migrasi control-plane 0002**: kolom `totp_secret` + `totp_enabled` (tenant lama otomatis ter-upgrade lewat runner migrasi).
- **Endpoint**: `POST /auth/2fa/setup` (rahasia + tautan otpauth), `/enable` dan `/disable` (keduanya wajib kode valid).
- **Alur login**: bila 2FA aktif, password benar saja tidak cukup — respons `twoFactorRequired` membuat form login menampilkan kolom kode 6 digit.
- **UI**: kartu **Keamanan** di Pengaturan (aktifkan: tampilkan kunci manual + tautan otpauth → konfirmasi kode; nonaktifkan juga butuh kode); status 2FA di `/me`.

### Landing page
- Hero baru, kisi 6 fitur unggulan, **seksi harga 3 paket** (selaras `PLAN_LIMITS`), footer — gaya SaaS bersih, siap menerima calon pengguna.

## Validasi (semua hijau)

- **20 unit test** (4 baru: TOTP dicek terhadap **vektor uji resmi RFC 6238**, verifikasi ±1 langkah, base32 bolak-balik).
- **Smoke test 88 pemeriksaaan** (8 baru): setup → aktivasi kode salah ditolak → aktivasi benar → login tanpa kode `twoFactorRequired` → kode salah 401 → password+kode benar 200 → nonaktifkan (kode dihitung independen di Node, membuktikan kompatibilitas lintas-implementasi).
- Playwright: landing page baru dirender benar (hero, fitur, harga).
