# Log Kerja — Fase 2i: Approval Engine Pembelian (Gelombang A-4)

**Tanggal:** 3 Juli 2026 · **Status akhir:** selesai, merged ke `main`

## Yang dibangun

- **Migrasi tenant 0006**: `approval_requests` — dokumen disimpan sebagai *payload* dan **baru diposting (jurnal + stok) saat Owner menyetujui**; tidak ada efek samping selama menunggu.
- **Ambang persetujuan** (khusus Owner, kartu di Pengaturan): pembelian oleh Admin dengan nilai ≥ ambang otomatis masuk antrean (202 + nomor `APR-xxxxx`); Owner sendiri bebas ambang.
- **Refactor** `executePurchase`: satu implementasi posting pembelian dipakai jalur langsung dan jalur persetujuan (payload divalidasi ulang saat approve — produk terarsip/periode terkunci tetap tertangkap).
- **Endpoint**: daftar permintaan (Owner), setujui (posting dieksekusi), tolak (dengan catatan); keduanya sekali-pakai; audit log lengkap (requested/approved/rejected/threshold_set).
- **UI**: halaman **Persetujuan** (Owner: setujui/tolak; non-owner: info), kartu ambang di Pengaturan, toast "menunggu persetujuan" di form pembelian.
